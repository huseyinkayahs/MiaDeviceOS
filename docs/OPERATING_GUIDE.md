# MiaDeviceOS Operating Guide

This guide explains how to operate and test the current MiaDeviceOS firmware.

## 1. Before running the device

Make sure this file exists locally:

```text
include/secrets.h
```

It must contain the real WiFi and MQTT settings.

Do not put real passwords inside:

```text
include/secrets.example.h
```

## 2. Build

In VS Code:

```text
PlatformIO > esp32dev > General > Build
```

Expected result:

```text
SUCCESS
```

## 3. Upload

Connect the ESP32 using USB.

Then run:

```text
PlatformIO > esp32dev > General > Upload
```

Expected result:

```text
Hard resetting via RTS pin...
SUCCESS
```

## 4. Monitor

Open:

```text
PlatformIO > esp32dev > Platform > Monitor
```

Expected startup:

```text
MiaDeviceOS v1.0.0
WiFi baglandi
MQTT baglaniyor... BAGLANDI
Config topic dinleniyor.
Command topic dinleniyor.
Heartbeat gonderildi
```

## 5. Basic health checklist

The device is healthy if these appear in Serial Monitor:

```text
WiFi baglandi
MQTT baglaniyor... BAGLANDI
Heartbeat gonderildi
Telemetry gonderildi
```

If WiFi fails:

```text
Check include/secrets.h
Check WiFi SSID/password
Upload again after saving secrets.h
```

If MQTT fails:

```text
Check MQTT_SERVER
Check MQTT_PORT
Check broker availability
```

## 6. MQTT command test

Use MQTT Explorer.

Publish to:

```text
mia/site01/laser01/command
```

Payload:

```json
{
  "command": "get_config",
  "request_id": "manual-test-001"
}
```

Expected Serial Monitor output:

```text
MQTT mesaj alindi. Topic: mia/site01/laser01/command
Command status gonderildi
```

Expected status topic:

```text
mia/site01/laser01/command/status
```

## 7. Config test

Publish to:

```text
mia/site01/laser01/config
```

Valid config:

```json
{
  "device_id": "laser01",
  "current_limit": 25,
  "temperature_limit": 30,
  "repeat_if_continues_min": 1,
  "normal_send_interval_sec": 10,
  "over_current_delay_sec": 5,
  "heartbeat_interval_sec": 30,
  "wifi_connect_timeout_sec": 15,
  "wifi_reconnect_interval_sec": 10,
  "mqtt_reconnect_interval_sec": 5
}
```

Expected result:

```text
Config basariyla uygulandi.
Config status gonderildi: status done
```

Invalid config example:

```json
{
  "device_id": "laser01",
  "current_limit": -5,
  "temperature_limit": 30
}
```

Expected result:

```text
Config reddedildi.
status: rejected
```

## 8. Alarm test

The simulated sensor currently reports:

```text
current: 20
```

To trigger alarm:

```text
Set current_limit below 20, for example 12.
```

Expected:

```text
ALARM_STARTED
```

To clear alarm:

```text
Set current_limit above 20, for example 25.
```

Expected:

```text
ALARM_CLEARED
```

## 9. Telegram notification test

n8n workflow must be published:

```text
MiaDeviceOS - Telegram Alarm Notification
```

Correct alarm flow:

```text
ESP32 → MQTT alarm topic → n8n → Telegram
```

Test method:

```text
1. Set current_limit to 25.
2. Telegram should receive alarm cleared.
3. Set current_limit to 12.
4. Telegram should receive alarm started.
```

Important:

```text
Do not publish fake alarm payloads manually for final validation.
Alarm events must be generated by ESP32.
```

## 10. OTA test

Keep the ESP32 and PC on the same WiFi network.

Find the PC IP:

```powershell
ipconfig
```

Example:

```text
192.168.1.2
```

Start HTTP server:

```powershell
cd "C:\New DeviceOs Project\MiaDeviceOS_v0.5_command_engine (1)\.pio\build\esp32dev"
& "$env:USERPROFILE\.platformio\penv\Scripts\python.exe" -m http.server 8000 --bind 0.0.0.0
```

Open in browser:

```text
http://192.168.1.2:8000/
```

You should see:

```text
firmware.bin
```

Then publish to:

```text
mia/site01/laser01/command
```

Payload:

```json
{
  "command": "ota_update",
  "request_id": "ota-local-test-001",
  "url": "http://192.168.1.2:8000/firmware.bin",
  "version": "1.0.0-test"
}
```

Expected Serial Monitor output:

```text
OTA baslatiliyor
OTA firmware boyutu
OTA basarili
OTA status gonderildi: status done
OTA sonrasi restart uygulaniyor
```

After restart:

```text
MiaDeviceOS v1.0.0
WiFi baglandi
MQTT BAGLANDI
Heartbeat gonderildi
```

## 12. Troubleshooting

### `pio` is not recognized

Use PlatformIO buttons inside VS Code instead of terminal commands.

### COM port access denied

Close all monitor terminals, unplug ESP32, wait 5 seconds, plug it back in, then reopen Monitor.

### WiFi shows `YOUR_WIFI_NAME`

You edited the wrong file. Create/edit:

```text
include/secrets.h
```

Not:

```text
include/secrets.example.h
```

### OTA HTTP GET failed

Check:

```text
HTTP server is still running
PC IP is correct
ESP32 and PC are on same WiFi
Windows Firewall is not blocking port 8000
```
