
# V3.6.4 Database Calculated Runtime Fix

Bu paket v3.6.3 üzerine kopyalanır.

## Sorun

Dashboard RUNNING iken doğru canlı süreyi gösteriyordu.
Fakat makine STOPPED olunca çalışma süresi tekrar eski daily_summary snapshot değerine dönüyordu.

Örnek:

```text
RUNNING iken: 4 sa 12 dk
STOPPED olunca: 56 sn
```

## Neden

`latest_state.duration_sec` aktif state'in süresidir.

```text
RUNNING iken latest_state.duration_sec = aktif çalışma süresi
STOPPED iken latest_state.duration_sec = aktif duruş süresi
```

Bu yüzden toplam günlük çalışma süresi sadece dashboard tarafında doğru hesaplanamaz.

## Düzeltme

Backend artık bugünkü runtime ve stop süresini PostgreSQL'deki `machine_state_events` tablosundan hesaplar.

Yeni API alanı:

```text
calculated_today_summary
```

İçerik:

```text
runtime_sec
stop_sec
observed_sec
utilization_pct
```

Dashboard artık şu alanı kullanır:

```text
st.calculated_today_summary
```

RUNNING iken runtime artar.
STOPPED yapınca runtime eski değere düşmez.
STOP süresi artmaya başlar.
Tekrar RUNNING yapınca runtime kaldığı yerden artar.
