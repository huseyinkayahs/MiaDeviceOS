
# V4.6 SaaS Foundation Pack

Bu paket v4.5.0 üzerine kopyalanır.

## Amaç

FactoryBox'ı tek atölye panelinden çok müşterili SaaS altyapısına hazırlamak.

Bu sprintte 3 sprint birleştirildi:

```text
1) User Login altyapısı
2) Customer / Tenant yapısı
3) Dashboard erişim ayrımı temeli
```

## Eklenenler

```text
Login ekranı
Auth API endpointleri
Token tabanlı API erişim temeli
Customer / Tenant context endpointleri
Site erişim kontrolü temeli
Dashboard SaaS Foundation kartı
Auth .env örnek dosyası
SaaS database migration dosyası
```

## Kurulum

ZIP'i proje ana klasörünün üzerine çıkar:

```text
C:\New DeviceOs Project\MiaDeviceOS_v0.5_command_engine (1)
```

"Dosyaları hedefte değiştir" seçeneğini onayla.

## Başlangıç Modu

Bu paket güvenli geçiş için varsayılan olarak auth kapalı gelir:

```text
AUTH_ENABLED=false
```

Böylece mevcut dashboard bozulmaz.

## Login'i Açmak İçin

Backend `.env` dosyana ekle:

```text
AUTH_ENABLED=true
FACTORYBOX_ADMIN_EMAIL=admin@factorybox.local
FACTORYBOX_ADMIN_PASSWORD=FactoryBox123!
FACTORYBOX_ADMIN_ROLE=owner
AUTH_SESSION_HOURS=12
```

İstersen admin email ve şifreyi değiştir.

## Backend Yeniden Başlatma

```powershell
cd "C:\New DeviceOs Project\MiaDeviceOS_v0.5_command_engine (1)\platform\backend"
npm.cmd start
```

Eğer port hatası olursa:

```powershell
netstat -ano | findstr :3100
taskkill /PID <PID> /F
npm.cmd start
```

## Önemli Not

Bu sprint production seviyesinde tam kullanıcı yetkilendirme değildir.

Bu sprintin amacı:

```text
SaaS mimarisinin ilk temelini atmak
çok müşteri / çok site ayrımı için altyapıyı hazırlamak
sonraki sprintlerde role-based access, tenant yönetimi ve kullanıcı ekranlarına geçmek
```
