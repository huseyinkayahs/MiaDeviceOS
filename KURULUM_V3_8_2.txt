
# V3.8.2 ai_reports Legacy Schema Compatibility Fix

Bu paket v3.8.1 üzerine kopyalanır.

## Sorun

Kayıt testinde şu hata görüldü:

```text
null value in column "summary_text" of relation "ai_reports" violates not-null constraint
```

Neden:

Eski veritabanı şemasında `ai_reports.summary_text` kolonu `NOT NULL` kalmıştı.
v3.8.1 yeni kayıtları `summary` kolonuna yazıyordu, ama `summary_text` boş kaldığı için PostgreSQL kayıt kabul etmedi.

## Düzeltme

Backend başlangıcında:

```text
summary_text kolonu varsa NOT NULL kaldırılır
report_text kolonu varsa NOT NULL kaldırılır
summary_text / summary boş kayıtlar doldurulur
```

Kayıt sırasında:

```text
summary
summary_text
report_text
```

alanlarının hepsi aynı özet metni ile doldurulur.


